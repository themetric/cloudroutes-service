HTTP(S) Request

Overview

Makes an HTTP(S) request via GET or POST.

Reaction Actions

This reaction will only perform a single reaction, this reaction can be configured to occur on "Failed Monitors" or "Healthy Monitors" but is not designed to be executed on both. In order to setup this reaction to have an action for "Healthy Monitors" and a different action for "Failed Monitors" you can create two reactions and attach them to the same Monitor.

Reaction Form Details

URL

This is the URL you would like tht HTTP(S) request to be sent to.

HTTP(S) Request Type

This is the type of HTTP(S) request you would like to use. GET requests will fetch some information from the URL while POST requests will post some the data from the HTTP body to the remote URL.

Continuous

If set to Yes, the Continuous option will tell this reaction to post the request continously.
